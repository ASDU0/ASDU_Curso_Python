# ¿Qué pasa cuando ejecutas código?
Cuando escribes código, la computadora no lo entiende directamente.
Tu código pasa por una "traducción" antes de convertirse en acciones reales.
Es como hablarle a alguien en otro idioma: necesitas un traductor en el medio.

# Compiladores vs Intérpretes
Existen dos formas de traducir código:
| | Compilador | Intérprete |
|---|---|---|
| ¿Cuándo traduce? | Todo de una vez, antes de ejecutar | Línea por línea, mientras ejecuta |
| Ejemplo | C, C++ | Python, JavaScript |
| Velocidad | Más rápido al ejecutar | Más flexible para aprender |
> Python usa intérprete → por eso puedes probar código al instante.
---
# El ciclo de un programa
Tú escribes código → Python lo interpreta → La CPU lo ejecuta → Ves el resultado
Cada línea que escribes sigue este camino, una por una.


# Resumen
Un programa es una secuencia de instrucciones que Python interpreta línea por línea.
El ciclo siempre es el mismo: **entrada → proceso → salida**.
